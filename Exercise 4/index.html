Exercise 4.1

Draw SVGs

A house and garden scene using all SVG shape types. The two windows are grouped using <g> so shared styling applies to both identically, and positioned with transform/translate.

📍

SVG Coordinate System

ℹ️

SVG coordinates start at (0,0) in the top-left corner. X increases rightward. Y increases downward.

▭ rect

House wall top-left at (200,230). Extends 300px right, 170px down.

● circle

Sun centre at (700,70). Radius 50 draws 50px in every direction.

▲ polygon

Roof: left eave (160,230) → peak (350,100) → right eave (540,230).

⬭ ellipse

Cloud centre (150,80). rx=60 wide, ry=30 short — flat cloud shape.

— line

Sun ray (700,10)→(700,0). Same x value = perfectly vertical line.

〰 polyline

Garden path: connected lines from door bottom (315,400) downward.

✦ path

Smoke: M moves to (430,125). Q draws quadratic bezier curve.

T text

Label at (350,490). text-anchor="middle" centres it around x.

⬡ g group

Both windows share one <g>. Each uses translate to position.

Exercise 4.5

Bind Data and Draw

Data is bound to SVG rectangles using selectAll / data / join. Bars are unscaled — width maps 1 count directly to 1 pixel so bars overflow the canvas. Scaling is added in the next exercise.

🔗

Unscaled Bars

⚠️

Bars overflow because there is no scaling yet — this is intentional to demonstrate the problem that scaling solves.

Exercise 4.6

Scaling Charts

d3.scaleLinear() maps count values to pixel widths. d3.scaleBand() divides the y-axis into equal rows for each brand with padding between bars.

📏

Exercise 4.7

Adding Labels

Each bar and its labels are placed in a <g> group element with transform/translate so they move together with the yScale. Brand names appear left, counts appear right of each bar.
