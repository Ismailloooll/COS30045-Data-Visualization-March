About This Project

COS30045 Data Visualisation — Semester 1, 2025 · Swinburne University of Technology

Project Overview

This website was built as part of the COS30045 Data Visualisation project at Swinburne University of Technology. Our brief was to design and implement interactive visualisations that allow users to explore the Bureau of Infrastructure and Transport Research Economics (BITRE) road safety enforcement dataset, and to tell a meaningful story about some aspect of that data.

The existing BITRE Police Enforcement Dashboard (built in Power BI) does not incorporate the new 2023 granular data fields — including location type (metropolitan/regional/rural/remote), age group, arrests, and charges. Our four visualisations directly address this gap by making those new dimensions explorable.

The Data

The dataset covers road safety enforcement activities across all Australian states and territories from 2008. Data is sourced from state and territory police agencies and state revenue offices and published annually by BITRE.

Key variables: Year (2008–2023), jurisdiction (all 8 states/territories), offence type (speeding, mobile phone use, drink driving, drug driving), and enforcement count. From 2023 onward: location type (metropolitan, regional, rural, remote) and age group of offender.

Source: BITRE Road Safety Enforcement Data 2024 — bitre.gov.au

Visualisations

We built four interactive visualisations using D3.js v7, each addressing a different dimension of the dataset:

  • Enforcement Trends Over Time — Multi-line chart (2008–2023), filterable by state/territory and offence type. Includes an indexed view (2008 = 100) to compare growth rates, and a year-on-year percentage change bar chart.
  • State & Territory Comparison — Grouped bar chart comparing all 8 jurisdictions for any selected year and offence. Includes a share-of-national-total horizontal bar chart, and click-to-highlight interaction.
  • Age Group Breakdown (2023) — Grouped and stacked bar charts using the new 2023 age-group data. Toggle between absolute and percentage display.
  • Location Type Analysis (2023) — Heatmap (location × offence), grouped bars by location, and animated donut charts per offence type using the new 2023 location data.

Technology Stack

D3.js v7 HTML5 CSS3 Vanilla JavaScript KNIME Analytics Platform GitHub Swinburne Mercury Server

Visualisations are built entirely in D3.js v7 with no additional charting libraries. All chart rendering is done client-side from CSV data files. Data processing and transformation was performed in KNIME Analytics Platform. Version control is managed via GitHub, and the site is hosted on Swinburne's Mercury server.

Design Decisions

The visual design uses a dark navy hero section to establish a professional, data-focused identity, with clean white chart containers that prioritise legibility. A consistent colour palette across all charts maps offence types to the same colours throughout the site (speeding = red, mobile phone = blue, drink driving = gold, drug driving = teal), reducing cognitive load when navigating between pages.

Interactive controls — dropdowns, click-to-highlight, toggle between display modes — were prioritised over static views to meet the brief's requirement to allow users to explore the data rather than just observe it. Tooltips provide exact values on hover for all data points across all charts.
